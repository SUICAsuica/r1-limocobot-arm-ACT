# Dataset Strategy: What Can Be Reused and What Must Be Collected

## 基本方針

本研究では、既存データセットが使える部分と、自作データが必要な部分を分けて考える。

災害環境の画像理解や物体認識には、既存の画像データセットや事前学習済みモデルを活用できる可能性がある。一方で、LIMO Pro2 + myCobot280の実機構成に固有の視覚と行動の対応は、自作データで集める必要がある。

## 既存画像データが使える可能性がある部分

以下には既存データや事前学習済みモデルが使える可能性がある。

- 人検出
- 瓦礫検出
- 物体認識
- セグメンテーション
- 危険領域の認識
- VLMによる一般的な画像理解
- ナビゲーションの事前学習

これらは、VLM、物体検出モデル、セグメンテーションモデル、ナビゲーションモデルの事前知識として利用できる。例えば「人らしき領域」「瓦礫らしき物体」「通れそうな空間」を認識する部分は、既存モデルを出発点にできる。

## 自作データが必要な部分

以下はmyCobot280 / LIMO Cobot構成に依存するため、自作データが必要である。

- myCobot280の関節角
- myCobotのグリッパー値
- LIMO Cobot上のカメラ視点
- 自分の机・照明・物体配置
- 把持するときのaction
- 車体＋アーム連携データ
- LIMO Pro2 + myCobot280特有の視覚と行動の対応

特にACTで学習するactionは、使用するロボットの関節構成、グリッパー、カメラ位置、制御周期に強く依存する。そのため、他のロボットの把持データをそのまま使うことは難しい。

## 表として整理

| 用途 | 既存データが使えるか |
|---|---|
| 瓦礫・人・物体の認識 | 使える |
| VLMによる状況理解 | 使える |
| ナビゲーションの事前学習 | 使える |
| myCobotの関節角action | 基本的に自作が必要 |
| LIMO Cobot特有の把持データ | 自作が必要 |
| 車体＋アーム連携データ | 自作が必要 |

## LeRobot形式で保存する理由

ACTで学習するためには、画像・関節角・gripper・actionを決まった形式で保存する必要がある。LeRobotDataset形式にすることで、ACTや他の模倣学習手法に使いやすくなる。

LeRobot形式では、画像は動画として保存し、数値データはparquetなどで保存する。actionは、現在状態から数フレーム先の目標関節角として作成する。

例:

```text
observation.images.rgb:
  LIMO車体RGBカメラ

observation.images.arm:
  アームカメラ

observation.state:
  [j1, j2, j3, j4, j5, j6, gripper]

action:
  [target_j1, target_j2, target_j3, target_j4, target_j5, target_j6, target_gripper]
```

この形式にそろえることで、データ収集、学習、評価、再利用の流れを整理しやすくなる。

## 2カメラの役割

2カメラ構成では、LIMO RGBカメラとarmカメラに異なる役割を持たせる。

```text
LIMO RGBカメラ:
  全体を見るカメラ
  物体の位置、作業範囲、グリッパー先端を確認する

armカメラ:
  手元を見るカメラ
  掴む直前の物体とグリッパーの位置関係を見る
```

両方に写った方がよいもの:

- 把持対象の物体
- グリッパー先端
- 作業範囲の一部

LIMO RGBカメラに写ってほしいもの:

- 物体
- 机上作業範囲
- グリッパー先端
- アームの接近方向

armカメラに写ってほしいもの:

- 把持対象
- グリッパー先端
- 接触しそうな部分

映らなくてよいもの:

- LIMO本体
- 背景の壁や部屋
- 人の手
- 関係ない物体

2つのカメラを使うことで、全体の作業状況と手元の細かい位置関係を同時に扱える。将来的にVLMが対象物を指定し、ACTがその対象物に対して操作を実行する場合にも、この視点分担が重要になる。
