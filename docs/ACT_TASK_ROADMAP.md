# ACT Task Roadmap for myCobot280

## なぜACTに全体タスクを学習させないのか

「崩落現場で人命救助」を1つのACTタスクとして学習させるのは難しい。災害環境では、探索、判断、移動、対象物選択、安全確認、再計画など、多くの判断が必要になるためである。

ACTには小さな操作スキルを学習させ、未知環境での判断はVLM/LLM/地図/ルールベースに任せる。

```text
VLM / LLM:
  見る・判断する・計画する

ACT:
  掴む・押す・引く・どかすなどの低レベル動作を実行する
```

つまり、ACTは「何をすべきか」を決めるモデルではなく、指定された低レベル操作を安定して実行するaction expertとして扱う。

## ACTに学習させるべき操作プリミティブ

| 段階 | タスク | ACTに学習させる内容 | 目安episode |
|---:|---|---|---:|
| 1 | 青い物体を掴む | 基本把持 | 30〜50 |
| 2 | 木片を掴む | 瓦礫に見立てた直方体把持 | 50〜100 |
| 3 | 木片を横にどかす | 把持 → 移動 → 解放 | 80〜150 |
| 4 | 物体を押してどかす | 掴めない物体への押し操作 | 50〜100 |
| 5 | 斜め・不規則な物体をどかす | ゴム製ドアストッパーや軽い瓦礫風物体 | 100〜200 |
| 6 | 人形の一部を避けて瓦礫をどかす | 接触してはいけない領域を避ける操作 | 150〜300 |
| 7 | 複数瓦礫から1つを選んでどかす | VLMが選んだ対象物に対する操作 | 200〜300 |
| 8 | 失敗回復 | 掴み損ねた後に再接近・再把持 | 100〜200 |

## 現実的な初期目標

初期段階では、災害環境全体を再現するのではなく、把持と除去に必要な小さな操作を順番に増やす。

| フェーズ | 内容 | episode数 |
|---|---|---:|
| Phase 1 | 青い物体を掴む | 30 |
| Phase 2 | 木片を掴む | 50 |
| Phase 3 | 木片を横にどかす | 100 |
| Phase 4 | 位置を3パターンに増やす | 150 |
| Phase 5 | 人形ダミーの近くの木片をどかす | 200 |

合計で約530 episodeを初期目標とする。

## データ収集の考え方

最初は多様性より一貫性を重視する。いきなり物体、位置、照明、接近方向を増やすと、ACTが何を学習すべきか分かりにくくなる。

最初の50 episodeは、次のような条件で収集する。

- 対象物: 同じ青い物体
- 位置: ほぼ同じ
- 初期姿勢: ほぼ同じ
- 近づき方: 基本は同じ方向
- 照明・カメラ: 固定
- データ: 成功デモのみ

ただし、完全に同じ軌道を50回繰り返すのではなく、自然な微小なズレを含む成功デモを集める。

```text
同じタスク・同じ方向・同じ条件で、
自然な微小なズレを含む成功デモを50回
```

左から向かう、右から向かうなどの大きなバリエーションは、基本把持が安定してから追加する。

## 評価方法

把持タスクは最終成功率だけでなく、サブタスクごとに評価する。どの段階で失敗しているかを分けることで、データ、モデル、実機制御のどこを改善すべきか判断しやすくなる。

| 評価項目 | 判定基準 |
|---|---|
| 接近成功 | グリッパーが物体近くまで到達 |
| 位置合わせ成功 | 物体を挟める位置に到達 |
| 把持成功 | グリッパーで物体を保持 |
| 持ち上げ成功 | 物体を机から浮かせる |
| 保持成功 | 0.5〜1秒落とさず保持 |
| 完全成功 | 把持から保持まで完了 |

実験ログには、成功/失敗だけでなく、どの評価項目まで到達したかを記録する。これにより、例えば「接近はできるが保持できない」「位置合わせはできるがグリッパーが開く」といった問題を切り分けられる。
